# SL:IT 랜딩페이지 SI 에이전시 리뉴얼 플랜

**Date:** 2026-05-29
**Status:** 방향 확정 / 구현 대기 (카피라이팅 추가 고민 필요)

## 1. 목적 / 포지셔닝

- **1순위 목적:** 외주 고객 유치 (SI·개발 에이전시 포지션). 레퍼런스 4곳(트리숲, 똑똑한개발자, 대모산개발단, 리트머스)과 동일한 목적.
- 기존 사이트는 "SL:IT — MVP·AI 서비스·SaaS 개발 에이전시"였으며 이 포지션을 강화한다.
- 채용/팀 페이지(`recruiting-team-page-plan.md`)는 별도 트랙으로 유지. 이 플랜은 고객용 메인 랜딩 전용.

## 2. 확정 의사결정

1. **1순위 타깃:** 외주 고객 유치.
2. **규모 포지셔닝:** 규모감 연출 ("개발자 많은 척"). 단, **검증 가능한 거짓 숫자는 금지**.
   - ❌ "팀원 N명", "30+ 프로젝트" 같이 검증되면 들통나는 거짓 수치.
   - ✅ 역량 그리드를 넓게, 포트폴리오 카테고리를 다양하게, 프로세스를 체계적으로, 1인칭 "저희 팀" 화법 → 자연스러운 규모감.
   - ✅ "AI-native라 소수가 다량을 처리한다"(트리숲 논리)로 속도=규모를 정당화.
   - **이유:** "정직·명확" 메시지(anti-외주PTSD)와 거짓 규모 주장이 충돌하면 고객 미팅에서 2인 팀임을 알게 되는 순간 브랜드 신뢰가 붕괴됨.
3. **히어로 전면 신뢰자산:** AI-native 속도 + 정직·명확(anti-외주PTSD).
4. **히어로 카피 방향:** C안(파트너 포지션) 방향. "레거시 산업에 AI를 넣는 개발 파트너" 류.
   - **카피라이팅은 추가 고민 필요 (확정 아님).**
   - **강점 메모:** lawclaw(법률 외주) 같은 실제 외주 수행 제품에 강점이 있음 → 포트폴리오/케이스에서 부각.

- **트리숲 AI-native 루프**: "Claude Code로 2~3배 빠름". 규모 약점을 강점으로 전환하는 핵심 무기.

## 4. 새 섹션 아키텍처

> 모노 팔레트·큰 한글 타이포·3카드·다크 섹션 리듬은 유지. 골격만 SI 신뢰형으로 재배열.

1. **히어로 (신뢰 우선)** — C안 카피 + 듀얼 CTA(`무료 상담`/`회사소개서`) + 수상 배지(경기 딥테크 대상·성균관대 AI+X 글로벌 대상)
2. **숫자 밴드** — 거짓 없는 카운터: 첫 배포 ~3분 / AI 개발 루프 2~3배 / 수상 2회 / 진행 제품 수
3. **Pain Agitation** (대모산) — "이런 외주 경험 없으셨나요" 4카드 → SL:IT 반박
4. **상품화 서비스 티어** (리트머스) — MVP 6주 / AI 기능 이식 / 운영 자동화 / Figma-to-Code
5. **AI-native 작업 방식** (트리숲) — 개발 루프 = 속도·규모감의 근거
6. **메트릭 포트폴리오** — Anyon·Bookface·Recova·lawclaw, 커밋·배포 숫자와 카테고리 라벨
7. **역량 그리드** — 넓게 펼쳐 규모감 (웹·앱·AI·자동화·SaaS·운영툴)
8. **차별점 / Why us** — 번호형
9. **후기 / 고객 학습 서사** — 실명 후기 확보 전이면 B2B 미팅 학습 서사로 대체
10. **FAQ** — "소규모인데 괜찮나" 정면 처리(정직 톤)
11. **강한 CTA** — 트러스트 불릿 + 회사소개서 + 상담

## 5. 비주얼 제약 (유지/회피)
