# AI · LLM 에이전트 개발

여러 모델과 채널, 운영 규칙이 함께 맞물리는 AI 제품을 실제 서비스 구조로 연결하는 역량입니다. 단순 챗봇 응답 생성보다, 프로젝트 성격에 따라 멀티 모델 라우팅, 역할 분리형 에이전트, 메모리, 페르소나, 보이스, 컴플라이언스 로직, 외부 서비스 연동, 운영 로그를 조합한 실행 시스템을 설계하고 구축하는 데 강점이 있습니다.

## 역량 개요

SL:IT는 LLM을 한 번 호출하는 기능보다, 여러 모델과 실행 계층을 묶어 제품으로 굴러가게 만드는 일에 익숙합니다. Anyon에서는 멀티 모델과 멀티 에이전트 구조를 실제 개발 플랫폼으로 운영했고, Lyla에서는 메모리와 규칙, 푸시, 전화 채널을 하나의 사용자 상태로 연결했습니다. Recova에서는 AI 보이스 엔진 위에 컴플라이언스 로직과 감사 체계를 얹었고, LawTeam TSS에서는 질문 분류, 스코어링, 페르소나 기반 답변 생성 파이프라인을 짧은 기간 안에 정리했습니다. Wireframe AI에서는 다중 LLM과 샌드박스 실행을 결합해 생성형 UI 흐름을 제품화했습니다.

## 우리가 잘하는 문제 유형

- 모델 하나로는 품질 편차가 커서, 여러 모델 제공자와 대체 경로 구조가 필요한 경우
- 역할별 에이전트를 나눠 기획, 생성, 검증, 실행 단계를 분리해야 하는 경우
- 사용자 맥락을 기억하는 메모리 구조나 규칙 기반 상태 관리가 필요한 경우
- 사람마다 다른 말투와 전문성을 반영한 페르소나 응답 체계가 필요한 경우
- 보이스 채널을 붙이되 금지 표현, 연락 제한, 감사 로그 같은 통제 장치가 함께 필요한 경우
- 단순 데모가 아니라 외부 서비스 연동, 저장, 검증, 배포까지 이어지는 AI 제품이 필요한 경우

## 구현 방식

1. 먼저 문제를 대화 생성 문제로만 보지 않고, 어떤 판단 단계와 운영 제약이 있는지부터 분해합니다.
2. 그 다음 단일 모델 고정 대신 멀티 모델 구조와 역할 분리형 에이전트 구조를 잡습니다.
3. 대화형 제품이라면 메모리, 규칙, 사용자 상태, 채널별 실행 경계를 따로 둡니다.
4. 규제 산업이나 민감 도메인이라면 사전 차단 로직과 사후 감사 로그를 함께 설계합니다.
5. 마지막으로 검증, 외부 연동, 배포까지 포함해 실제 운영 가능한 형태로 마무리합니다.

## 대표 프로젝트 레퍼런스

### Anyon

- OpenAI, Anthropic, Bedrock, Google, xAI 계열 SDK와 MCP SDK를 함께 연결한 멀티 모델 실행 구조를 운영했습니다.
- 멀티 에이전트 개발 플랫폼으로 설계해 기획, 구현, 외부 서비스 연동, 배포 흐름을 하나의 제품 안에서 다뤘습니다.
- Playwright 자동화와 결합해 외부 서비스 설정과 배포 검증까지 이어지는 AI 실행 환경을 만들었습니다.

### Recova

- 아웃바운드 AI 보이스 엔진 위에 7일 rolling 카운터, 야간 차단, 제3자 안전 모드, 금지표현 필터, 소멸시효 체크, 고지의무 검증을 결합한 컴플라이언스 구조를 설계했습니다.
- 통화 후 STT 감사 로그를 남기는 운영 흐름을 포함해, 규제 산업 도입을 전제로 검토할 수 있는 자동화 제품 구조를 만들었습니다.

### LawTeam TSS

- 질문 수집, 분류, 수임확률 스코어링, 변호사 배정, 페르소나 기반 답변 생성, 자동 게시까지 이어지는 멀티 에이전트 파이프라인을 구성했습니다.
- 변호사 4인의 전문분야와 말투를 페르소나 자산으로 분리해 재사용 가능한 응답 시스템으로 정리했습니다.

### Lyla

- 채팅, 규칙, 메모리, 관계 단계, 루틴, 푸시, 전화가 같은 사용자 상태를 공유하도록 설계했습니다.
- OpenAI와 Anthropic 이중 경로, 메모리 추출 로직, node-cron 기반 스케줄링을 결합해 개인 비서형 AI 제품 구조를 만들었습니다.

### Wireframe AI

- Vercel AI SDK 기반 다중 LLM 지원 구조를 두고, 자연어 입력에서 와이어프레임과 프로토타입 초안 생성까지 이어지는 흐름을 구현했습니다.
- E2B SDK 샌드박스 실행과 결과 렌더링을 연결해 생성형 도구를 실제 서비스형 제품으로 확장했습니다.

## 사용할 기술 예시

- OpenAI, Anthropic, Amazon Bedrock, Google, xAI
- Vercel AI SDK, MCP SDK
- Node.js, TypeScript, Express, Next.js, Electron
- Supabase, PostgreSQL, SQLite, Drizzle ORM
- STT/TTS 후보 비교 구조, SSE 스트리밍, node-cron
- Playwright, E2B SDK

## 클라이언트가 얻는 가치

- 모델 데모를 넘어서, 멀티 모델 라우팅, 에이전트 역할 분리, 외부 API 연결, 운영 로그까지 포함한 AI 제품 구조를 비교적 이른 단계부터 설계할 수 있습니다.
- 멀티 모델 구조 덕분에 특정 제공자 장애나 정책 변화에 대비한 대체 경로를 초기에 함께 검토할 수 있습니다.
- 메모리, 페르소나, 보이스, 규칙 엔진, 컴플라이언스 레이어를 분리해 두기 때문에 기능 추가와 운영 통제 기준을 문서화하기 좋습니다.
- 규제나 민감 도메인에서는 감사 로그, 금지 표현 통제, 연락 제한 같은 요구사항을 제품 구조 안에 함께 반영할 수 있습니다.
- 외부 서비스 연동, 자동화, 배포 검증 흐름까지 한 범위에서 설계해 도입 이후 운영팀이 인수할 수 있는 형태로 넘기기 수월합니다.
