# 빠른 MVP 설계 · 구축

짧은 시간 안에 제품의 핵심 가설을 검증할 수 있는 MVP를 설계하고, 실제로 동작하는 수준까지 빠르게 끌어올리는 역량입니다. 속도만 내는 방식이 아니라, 이후 확장 가능성을 남기는 구조로 처음 버전을 만드는 데 강점이 있습니다.

## 역량 개요

SL:IT는 아이디어를 설명서에만 남겨두지 않고, 바로 써볼 수 있는 제품 형태로 바꾸는 일을 반복해왔습니다. Anyon은 기획부터 개발, 연동, 배포까지 한 흐름으로 묶인 AI 개발 플랫폼으로 발전했고, TREEDI는 컨텍스트 오염이라는 문제를 새로운 인터페이스 구조로 실험했습니다. Wireframe AI는 자연어 입력만으로 와이어프레임과 프로토타입 초안을 생성하는 서비스로 확장됐고, Founder Sprint는 배치 운영 현장의 반복 업무를 빠르게 웹 시스템으로 옮긴 외주형 MVP 사례였습니다.

## 우리가 잘하는 문제 유형

- 아이디어는 명확하지만 첫 버전을 어디까지 만들어야 할지 정리가 안 된 경우
- 빠른 검증이 중요하지만, 단순 목업이 아니라 실제 사용 가능한 프로토타입이 필요한 경우
- 사용자 인터뷰와 관찰을 바탕으로 문제 정의를 빠르게 제품 구조에 반영해야 하는 경우
- 데스크톱 앱, 웹 앱, 생성형 도구처럼 제품 형태가 다양해도 짧은 주기로 실험해야 하는 경우
- 초기 구축 후 바로 운영하거나 외주 납품으로 이어져야 하는 경우

## 구현 방식

1. 가장 먼저 검증할 핵심 문제와 사용자 흐름을 정리합니다.
2. 기능을 한 번에 크게 만들기보다, 제품 가설을 검증하는 최소 단위로 잘라 MVP 범위를 잡습니다.
3. React, Next.js, Electron, Supabase 같은 검증된 조합으로 빠르게 첫 버전을 만듭니다.
4. 인터뷰, 사용 패턴, 운영 피드백으로 다음 사이클 우선순위를 조정합니다.
5. 초기에 만든 구조가 버려지지 않도록 도메인, 저장, 실행 계층을 점진적으로 정리합니다.

## 대표 프로젝트 레퍼런스

### Anyon

- 사용자 인터뷰 56명과 B2B 인터뷰 12곳을 바탕으로, 비개발자와 기업이 실제로 배포 가능한 결과물을 만들 수 있는 개발 플랫폼을 설계했습니다.
- Electron 앱, 서버, 워커, 템플릿, 외부 연동, 배포 흐름을 점진적으로 확장하며 제품 완성도를 높였습니다.

### TREEDI

- 기존 채팅형 AI의 컨텍스트 오염 문제를 트리 구조 인터페이스로 재정의하고, 웹과 Electron 위젯을 병행 실험했습니다.
- 20명 인터뷰와 100명 관찰을 바탕으로 문제 공감도를 검증한 뒤 제품 실험으로 이어간 사례입니다.

### Wireframe AI

- 자연어 설명만으로 와이어프레임과 프로토타입 초안을 생성하는 핵심 흐름을 먼저 만들고, 이후 샌드박스 실행, 상태 저장, 결제 시스템까지 확장했습니다.
- 빠른 시안 검토라는 명확한 문제를 생성형 UI 서비스로 연결했습니다.

### Founder Sprint

- 배치 운영 현장에서 반복적으로 발생하는 관리 업무를 짧은 기간 안에 하나의 운영 웹으로 통합했습니다.
- 실운영 조직이 바로 사용할 수 있는 관리자형 시스템으로 납품된 외주 사례입니다.

## 사용할 기술 예시

- React, Next.js, TypeScript, Tailwind CSS, Electron
- Supabase, PostgreSQL, SQLite, Prisma
- OpenAI, Anthropic, Vercel AI SDK
- Excalidraw, D3.js, E2B SDK
- Playwright, Vercel

## 클라이언트가 얻는 가치

- 아이디어를 빠르게 실제 화면과 동작으로 검증할 수 있습니다.
- 초기 버전부터 사용자 흐름과 운영 흐름을 함께 고려해 다시 만드는 비용을 줄일 수 있습니다.
- 웹, 데스크톱, 생성형 UI 등 제품 형태가 달라도 검증 속도를 높일 수 있습니다.
- 인터뷰와 실제 사용 맥락을 반영한 MVP를 만들 수 있어, 다음 투자나 외주 확장, 내부 운영 전환에 유리합니다.
- 빠르게 만들면서도 이후 확장을 버틸 수 있는 구조를 함께 확보할 수 있습니다.
