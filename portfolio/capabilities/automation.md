# 운영 자동화 · 외부 연동 시스템

반복 업무를 사람 손에서 떼어내고, 수집, 판단, 실행, 검증까지 이어지는 자동화 흐름을 만드는 역량입니다. 단순 스크립트 작성보다, 운영팀이 실제로 의존할 수 있는 자동화 시스템과 외부 서비스 연결 구조를 설계하고 구현하는 데 초점을 맞춥니다.

## 역량 개요

SL:IT는 자동화를 보조 기능이 아니라 운영 구조 자체로 다룹니다. Founder Sprint에서는 배치 운영과 팀 관리 업무를 웹 시스템으로 옮겼고, LawTeam TSS에서는 리드 수집부터 답변 게시까지 이어지는 자동 파이프라인을 구성했습니다. Recova에서는 보이스 자동화에 규제 통제와 감사 로그를 결합했고, Anyon에서는 외부 서비스 설정과 배포 검증을 Playwright 자동화로 연결했습니다. 즉, 수집, 분류, 연동, 배포, 감사가 필요한 운영 흐름을 제품 안에 녹여내는 쪽에 강합니다.

## 우리가 잘하는 문제 유형

- 반복 운영 업무가 스프레드시트, 메신저, 수작업에 흩어져 있는 경우
- 크롤링과 정제, 우선순위 판단, 후속 액션이 하나의 파이프라인으로 이어져야 하는 경우
- 이메일, 캘린더, 외부 협업 도구, 배포 환경 같은 외부 서비스 연동이 필요한 경우
- 규제 산업처럼 자동화와 함께 통제 로직, 기록 보존, 사후 감사가 필요한 경우
- 설정 작업까지 자동화해 도입과 운영 비용을 줄여야 하는 경우

## 구현 방식

1. 자동화 대상이 되는 운영 단계를 수집, 판단, 실행, 기록 단계로 나눕니다.
2. 사람이 꼭 판단해야 하는 지점과 시스템이 대신할 수 있는 지점을 먼저 구분합니다.
3. 크롤러, 스케줄러, 외부 API 연동, 배포 자동화, 검증 자동화를 각각 독립 계층으로 둡니다.
4. 규제나 민감 업무라면 차단 규칙과 감사 로그를 기본 구조에 포함합니다.
5. 마지막에는 운영자가 추적할 수 있도록 관리자 화면, 문서 자산, 로그 체계까지 함께 정리합니다.

## 대표 프로젝트 레퍼런스

### Founder Sprint

- 배치 프로그램 운영에 필요한 스타트업, 팀, 진행 상황, 과제, 세션, 오피스아워, 피드 흐름을 하나의 웹으로 통합했습니다.
- Nodemailer와 Google APIs를 활용해 초대와 일정 관련 운영 업무를 자동화했습니다.
- 반복 운영을 스프레드시트와 문서 중심 방식에서 웹 기반 시스템으로 옮긴 사례입니다.

### LawTeam TSS

- 로톡 질문을 수집하고, 분류와 수임확률 스코어링을 거쳐 적합한 변호사에게 연결한 뒤 페르소나 답변을 자동 생성하고 게시하는 흐름을 설계했습니다.
- `scripts/crawlers/` 구조를 두어 크롤러 확장성을 남겼고, 법무법인별 포크 운영도 고려했습니다.

### Recova

- 아웃바운드 보이스 자동화에 7일 rolling 카운터, 야간 차단, 제3자 안전 모드, 금지표현 필터, 소멸시효 체크, 고지의무 검증을 결합했습니다.
- 통화 종료 후 STT 감사 로그를 남기는 구조로, 자동화 이후의 사후 점검까지 고려했습니다.

### Anyon

- Playwright 자동화를 활용해 외부 서비스 설정과 배포 전후 검증 흐름을 제품 경험 안에 연결했습니다.
- 여러 외부 서비스와 AI 제공자 연동을 하나의 개발 플랫폼 안에서 다룰 수 있게 구성했습니다.

## 사용할 기술 예시

- Playwright, Python 크롤러, TypeScript 스크립트
- Next.js, Node.js, Bun
- Google APIs, Nodemailer, Supabase, PostgreSQL
- OpenAI, Anthropic, MCP SDK
- STT 기반 감사 로그 구조, 규칙 기반 상태 머신, LLM 기반 대화 제어
- Vercel 배포 연동, 운영 로그 문서화

## 클라이언트가 얻는 가치

- 반복 업무를 시스템으로 옮겨 운영팀의 수작업과 누락을 줄일 수 있습니다.
- 크롤링, 분류, 응답, 알림, 배포 같은 흐름을 끊기지 않는 하나의 자동화 파이프라인으로 만들 수 있습니다.
- 외부 서비스 연동이 늘어나도 계층을 나눠 관리하므로 유지보수가 쉬워집니다.
- 규제나 민감 도메인에서도 기록과 통제를 포함한 자동화 구조를 설계할 수 있습니다.
- 도입 이후 운영자가 추적 가능한 로그와 관리 체계를 함께 가져갈 수 있습니다.
