# SL:IT

AI · 웹 풀스택 개발 에이전시. 비개발자 사업가, 스타트업, 기업 운영팀, 자동화가 필요한 조직을 위해 제품 개발과 운영 시스템 구축을 함께 맡습니다.

## SL:IT 소개

SL:IT는 화면 제작만 맡는 외주사가 아니라, 제품 설계부터 구현, 운영 자동화까지 한 흐름으로 연결하는 개발 에이전시입니다. 빠른 MVP가 필요한 초기 제품, 반복 업무를 줄여야 하는 운영 조직, AI를 실제 서비스에 붙여야 하는 팀과 주로 일합니다.

우리는 요구사항을 기능 목록으로만 받지 않습니다. 실제 운영에서 병목이 생기는 구간, 사람이 반복하는 업무, 출시 뒤 유지보수 비용이 커질 지점을 먼저 정리한 뒤 개발 범위를 설계합니다. 그래서 AI 기능, 웹 제품, 관리자 도구, 외부 연동 시스템을 하나의 실행 체계 안에서 묶는 프로젝트에 강합니다.

- [팀 개요 보기](team/overview.md)
- [파트너 네트워크 보기](team/partners.md)

## 우리가 만드는 것

- **AI · LLM 에이전트 개발**: 멀티 모델, 멀티 에이전트, 보이스, 규칙 엔진, 감사 가능한 AI 시스템 구축
- **풀스택 웹 · 제품 개발**: 고객용 서비스와 관리자 도구를 함께 갖춘 실운영형 웹 제품 개발
- **운영 자동화 · 외부 연동 시스템**: 반복 업무를 수집, 판단, 실행, 기록 흐름으로 자동화
- **빠른 MVP 설계 · 구축**: 핵심 가설 검증에 필요한 범위를 빠르게 설계하고 실제 동작 수준까지 구현

기술 스택과 구현 방식은 [기술 스택 문서](assets/tech-stack.md)에서 더 자세히 볼 수 있습니다.

## 핵심 실적 요약

- **누적 외주 매출 ₩3,700만**
- **수상 2건**, 중소벤처기업부 장관상 포함
- **9개 이상의 프로젝트 포트폴리오** 보유
- AI 제품, 운영형 웹, 관리자 시스템, 외부 연동 자동화까지 프로젝트 문서와 구현 범위로 확인 가능한 레퍼런스 축적

짧게 만들고 끝나는 프로젝트보다, 포트폴리오에 포함된 사례들처럼 실제 운영 시나리오를 반영한 제품과 시스템을 중심으로 작업해온 경험이 강점입니다. 초기 구축 이후에도 관리자 기능, 운영 흐름, 후속 확장을 감안한 구조를 함께 설계합니다.

## 대표 프로젝트

### 1. [Anyon](projects/anyon.md)

비개발자와 기업이 아이디어 구상부터 배포까지 이어갈 수 있게 설계한 AI 자동화 개발 플랫폼입니다. 멀티 모델, 멀티 에이전트, 배포 자동화를 하나의 제품 경험으로 묶은 사례입니다.

### 2. [Recova](projects/recova.md)

NPL 시장을 위한 AI 채권추심 보이스 에이전트이자 컴플라이언스 운영 시스템입니다. 보이스 자동화에 규제 통제와 감사 구조를 함께 반영한 사례입니다.

### 3. [SPEC Web](projects/spec-web.md)

비개발자 운영팀도 직접 관리할 수 있도록 설계한 실운영형 커뮤니티 플랫폼입니다. 브랜딩 사이트와 내부 운영 도구를 운영 시나리오에 맞춰 하나의 웹 제품으로 통합했습니다.

### 4. [Founder Sprint](projects/founder-sprint.md)

배치 운영, 스타트업 관리, 팀 협업, 진행 상황 추적을 하나의 웹으로 통합한 운영 자동화 시스템입니다. 스프레드시트와 수작업 중심 운영을 웹 기반 체계로 전환했습니다.

### 5. [LawTeam TSS](projects/lawteam-tss.md)

법률 상담 리드를 자동 수집하고, 수임 가능성 판단과 변호사별 답변 생성까지 연결한 멀티 에이전트 응답 시스템입니다. 짧은 구축 기간 안에 도메인 구조화와 자동화 파이프라인을 함께 설계했습니다.

더 많은 사례는 `projects/` 폴더의 상세 문서에서 확인할 수 있습니다.

## 작업 방식

작업은 보통 문제 정의, 범위 압축, 핵심 구현, 운영 연결의 순서로 진행합니다. 먼저 지금 해결해야 할 문제와 이후 단계로 미뤄도 되는 범위를 구분합니다. 그다음 핵심 사용자 흐름을 빠르게 구현하고, 출시 뒤 반복될 운영 작업까지 함께 고려해 자동화 포인트를 설계합니다.

기술 선택은 유행보다 유지보수성과 확장 가능성을 우선합니다. 복잡한 구조를 과하게 가져가기보다 팀 상황에 맞는 단순한 구조를 먼저 제안하고, 필요할 때 단계적으로 확장할 수 있게 만듭니다.

## 함께 일하는 이유

- 개발만 끝내는 것이 아니라, 실제 운영에서 필요한 관리자 기능과 자동화 흐름까지 함께 설계합니다.
- AI 기능도 단순 응답 데모에 머무르지 않도록, 프로젝트 성격에 따라 모델 연결, 실패 대응 경로, 외부 서비스 연동, 운영 로그까지 포함한 구조로 설계합니다.
- 비개발자 사업가와 운영팀도 이해하고 인수할 수 있는 결과물을 남깁니다.
- 빠르게 만들되 다시 뜯어고치지 않도록 초기 구조를 신중하게 잡습니다.
- 외부 파트너 네트워크와 함께 도메인 해상도와 실무 적용성을 높입니다.
