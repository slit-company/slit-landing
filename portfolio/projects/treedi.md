# TREEDI

질문 흐름을 선형 채팅에서 트리 구조 탐색으로 바꾼 AI 학습 인터페이스 실험

## 개요

TREEDI는 AI와의 대화를 단순한 채팅 로그가 아니라 탐색 가능한 지식 구조로 바꾸기 위해 만든 트리 기반 학습 도구다. 질문과 답변이 이어질수록 맥락이 섞이고, 이전 대화로 다시 돌아가기가 어려워지는 기존 인터페이스의 한계를 해결하는 데 초점을 맞췄다. 리포지토리 기준으로 2025년 9월 23일부터 2025년 10월 24일까지 개발이 이어졌고, React 기반 웹 애플리케이션과 Electron 위젯, Supabase 저장 구조, OpenAI 연동을 함께 실험한 제품이다.

## 클라이언트 과제

AI를 학습 보조 도구로 쓰는 사용자는 한 답변을 읽는 순간 여러 갈래의 추가 질문이 생긴다. 그러나 일반적인 채팅 인터페이스에서는 새 질문을 기존 맥락 위에 이어 붙여야 하므로, 서로 다른 탐색 흐름이 한 대화에 뒤섞이며 컨텍스트 오염이 발생한다. 이 문제는 답변 정확도뿐 아니라 사용자의 인지 부담으로도 이어진다. 질문을 잘 관리하는 일이 학습 자체보다 더 피곤해지는 구조였기 때문이다.

인터뷰 20명 중 18명이 이 문제에 공감했고, 100명 관찰 중 82명이 실제로 AI를 학습에 활용하고 있다는 점도 확인되었다. 즉, 과제는 새로운 채팅 앱을 만드는 것이 아니라, 학습 맥락을 보존하면서 여러 탐구 경로를 자연스럽게 다룰 수 있는 인터페이스를 만드는 일이었다.

## 솔루션

이 프로젝트에서는 각 질문과 답변을 트리 노드로 취급하고, 하나의 응답에서 파생되는 여러 궁금증을 별도 브랜치로 분기할 수 있도록 설계했다. 사용자는 한 주제를 깊게 파고드는 동시에, 다른 방향의 탐구를 독립된 컨텍스트로 유지할 수 있다. 이렇게 하면 기존 선형 채팅에서 자주 발생하던 맥락 충돌을 줄이면서, 학습 과정을 시각적으로 되짚고 재구성할 수 있다.

제품 경험은 웹 버전과 Electron 위젯 버전으로 병행 실험되었다. 웹에서는 트리 탐색과 콘텐츠 읽기 경험을 안정적으로 다듬고, Electron 위젯에서는 다른 작업 화면 위에서 단축키로 즉시 호출하는 사용 패턴을 검증했다. 특히 컨텍스트 오염 문제를 인터페이스 구조 자체로 해결하려 했다는 점이 핵심이다. 기술적으로는 D3 기반 트리 시각화, OpenAI 연동, 저장 구조, 렌더링 레이어를 조합해 질문 생성, 분기, 복귀, 재탐색이 하나의 흐름으로 이어지게 만들었다.

## 아키텍처

리포지토리는 `src/`, `electron/`, `server/`, `supabase/`, `api/`, `prompts/`를 중심으로 구성된다. `src/` 내부에는 `features/`, `domain/`, `infrastructure/`, `views/`, `components/`, `shared/`, `theme/`가 분리되어 있어, UI 레이어와 도메인 로직, 공통 인프라를 한 파일에 몰아넣지 않고 역할별로 나누려는 방향이 읽힌다. 이는 빠른 프로토타이핑 과정에서 복잡도가 커지는 문제를 줄이고, 기능별 변경을 비교적 분리해서 다루기 위한 구조로 볼 수 있다.

`electron/`은 메인 프로세스, 프리로드, 서비스, 트레이, 접근성 보조 기능을 담당하며 데스크톱 위젯 레이어를 구성한다. `server/agent-proxy/`와 `api/agent/`는 모델 호출과 중계 흐름을 보조하고, `supabase/`는 저장 및 스키마 관리 계층을 맡는다. `prompts/` 디렉터리는 생성형 기능의 입력 자산을 별도로 관리하는 역할을 한다. 전체적으로 TREEDI는 웹 인터페이스, 시각화 엔진, Electron 오버레이, AI 호출, 저장 계층을 느슨하게 분리해, 탐색형 학습 경험을 실험하기 위한 제품 구조를 갖췄다.

## 핵심 구현 하이라이트

- D3 트리 시각화를 적용해 질문과 답변의 분기 구조를 화면에서 바로 탐색할 수 있게 구현
- 하나의 응답에서 여러 후속 질문을 독립 브랜치로 확장해 컨텍스트 오염 문제를 구조적으로 완화
- Electron 위젯 형태를 함께 실험해 다른 작업 중에도 단축키로 호출 가능한 사용 시나리오 검증
- 웹 버전과 데스크톱 버전을 병행 운영하며 학습 도구로서의 접근성과 몰입감을 함께 테스트
- Markdown, 수식, 코드 하이라이트 렌더링을 붙여 설명형 학습 콘텐츠 소비 경험 강화
- 빠른 프로토타이핑 이후 도메인, 인프라, 기능 단위로 구조를 나누며 유지보수 가능한 형태로 정리
