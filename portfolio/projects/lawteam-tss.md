# LawTeam TSS

법률 상담 리드를 자동 수집하고 변호사별 답변까지 연결한 로톡 전용 멀티 에이전트 응답 시스템

## 개요

LawTeam TSS는 더신사 법무법인을 위해 설계한 로톡 자동 답변 시스템이다. 로톡에 올라오는 법률 질문을 수집한 뒤, 질문을 분류하고 수임 확률을 스코어링한 다음, 적합한 변호사를 배정하고 페르소나 기반 답변을 생성해 자동 게시하는 파이프라인으로 구성했다. 리포지토리 기준 개발 기간은 2026년 4월 15일 하루이며, 총 12 commits가 집중적으로 쌓였다. 짧은 구축 기간 안에 멀티 에이전트 설계, 도메인 지식 구조화, 크롤러 확장성까지 한 번에 정리한 프로젝트다.

## 클라이언트 과제

법무법인 입장에서는 로톡 질문이 곧 신규 상담 리드이지만, 모든 질문을 사람이 실시간으로 읽고 우선순위를 가려 답변하기는 어렵다. 질문 도메인도 형사, 이혼, 부동산, 의료, 군형사처럼 넓고, 같은 사안이라도 어느 변호사의 톤과 전문성이 더 적합한지 판단해야 한다. 결국 필요한 것은 단순 자동 댓글 도구가 아니라, 리드 수집부터 수임 가능성 판단, 담당 변호사 선택, 페르소나 일관성 있는 답변 생성까지 연결되는 자동화 체계였다.

## 솔루션

이 프로젝트에서는 로톡 질문 수집 -> 분류 -> 수임확률 스코어링 -> 변호사 배정 -> 페르소나 답변 생성 -> 자동 게시의 전체 흐름을 하나의 파이프라인으로 설계했다. `personas/`에는 변호사 4인의 말투와 전문분야를 반영한 페르소나 자산을, `knowledge/`에는 더신사 도메인 지식과 프로필 데이터를, `skills/`에는 답변 생성 규칙을, `scripts/crawlers/`에는 수집 자동화를 배치했다. 이 구조 덕분에 법무법인별로 포크해 재사용할 수 있고, 새로운 변호사나 신규 채널이 생겨도 파이프라인을 크게 바꾸지 않고 확장할 수 있다.

## 기술 스택

| 영역 | 사용 기술 |
| --- | --- |
| Runtime | Bun |
| Language | TypeScript |
| Persona System | `personas/`, `knowledge/profiles/` |
| Agent Prompting | `skills/write-answer.md` |
| Crawling / Automation | `scripts/validate.ts`, `scripts/crawlers/lawtalk.py` |
| Documentation | `docs/plan.md`, `docs/task.md`, `docs/sys_arch.md` |

## 아키텍처

리포지토리는 `personas/`, `knowledge/`, `skills/`, `scripts/`, `docs/`를 축으로 분리한 구조다. `personas/`는 장휘일, 남희수, 김연주, 정준현 변호사 4인의 페르소나 자산을 관리하고, `knowledge/`는 더신사 법무법인 공용 지식과 변호사별 YAML 프로필을 저장한다. `skills/`는 답변 생성 에이전트의 시스템 프롬프트를 담아 응답 품질 기준을 고정하고, `scripts/crawlers/`는 로톡 질문 수집과 향후 네이버 지식인 확장 여지를 준비한다. `docs/`는 계획, 작업 분담, 시스템 아키텍처를 문서화해 납품형 프로젝트에서도 유지보수 인수인계가 가능하도록 설계했다. 전체적으로는 데이터 수집, 판단, 페르소나 생성, 게시를 분리한 가벼운 멀티 에이전트 구조다.

## 핵심 구현 하이라이트

- 로톡 질문을 24시간 수집하고 후속 처리까지 연결하는 자동 응답 파이프라인 설계
- 질문 분류 이후 수임확률 스코어링 단계를 넣어 모든 리드를 동일하게 처리하지 않도록 우선순위화
- 장휘일, 남희수, 김연주, 정준현 변호사 4인 체계를 페르소나 자산으로 분리해 전문분야와 톤을 시스템 안에 반영
- `knowledge/`와 `profiles/` 구조를 통해 법무법인 도메인 지식과 변호사 속성을 재사용 가능한 자산으로 정리
- `scripts/crawlers/lawtalk.py` 중심 구조를 두고 이후 채널 확장을 고려한 크롤러 계층 준비
- `skills/write-answer.md`와 검증 스크립트를 통해 답변 생성 품질과 파일 구조 일관성을 유지하도록 설계
