# Wireframe AI

자연어 설명만으로 와이어프레임과 프로토타입 초안을 생성하는 AI 프로토타이핑 시스템

## 개요

Wireframe AI는 제품 아이디어를 텍스트로 설명하면 즉시 화면 구조와 프로토타입 초안으로 전환해주는 생성형 UI 제작 도구다. 기획자가 머릿속에 있는 요구사항을 디자인 툴로 다시 옮기는 시간을 줄이고, 대화 기반 인터페이스만으로 빠르게 시안을 검토할 수 있도록 설계했다. 리포지토리 기준 개발 기간은 2024년 7월 10일부터 2026년 1월 22일까지다. 단순 데모를 넘어 생성 파이프라인, 샌드박스 실행, 상태 저장, 결제 시스템까지 포함한 서비스형 구조로 발전한 프로젝트다.

## 클라이언트 과제

초기 제품 기획 단계에서는 아이디어를 시각 구조로 옮기는 속도가 의사결정 속도를 좌우한다. 하지만 기존 방식은 기획 문서 작성, 화면 러프 스케치, 프로토타입 구성, 검토 과정을 각각 다른 도구에서 수행해야 해서 전환 비용이 컸다. 클라이언트가 원한 것은 자연어 입력만으로 화면 초안을 빠르게 만들고, 여러 모델의 응답을 비교하면서 원하는 결과를 고도화할 수 있는 흐름이었다. 동시에 실제 서비스로 운영하려면 생성 실행 환경의 안정성, 사용자 상태 관리, 과금 처리까지 함께 풀어야 했다.

## 솔루션

이 프로젝트에서는 자연어 입력을 받아 와이어프레임과 프로토타입 결과물로 이어지는 end to end 생성 흐름을 설계했다. Next.js 14 기반 UI 위에 Vercel AI SDK를 연결해 다중 LLM 지원 구조를 만들고, 사용자 요청에 따라 서로 다른 모델을 선택하거나 비교할 수 있게 했다. 생성된 산출물은 Excalidraw 기반 렌더링으로 바로 확인할 수 있으며, 더 복잡한 실행은 E2B SDK 기반 샌드박스 환경에서 안전하게 처리하도록 구성했다. 데이터와 상태는 Supabase, Upstash, Vercel KV를 조합해 관리했고, Stripe를 연결해 유료 사용 시나리오까지 포함한 제품 구조를 완성했다.

## 아키텍처

리포지토리는 `app/`, `components/`, `lib/`, `sandbox-templates/`, `source/` 중심으로 구성된다. `app/`은 채팅 기반 입력 화면, 결과 미리보기, 결제와 계정 관련 라우트를 포함하는 애플리케이션 레이어이고, `components/`는 생성 결과를 시각적으로 제어하는 UI 모듈과 공통 인터랙션 컴포넌트를 담당한다. `lib/`는 모델 호출, 상태 처리, 저장 계층, 결제 연동 로직을 묶는 서비스 레이어 역할을 하며, `sandbox-templates/`는 샌드박스 안에서 실행될 템플릿 자산을 관리한다. `source/`는 생성 파이프라인과 렌더링 관련 리소스를 정리하는 레이어로 볼 수 있다. 이 구조 덕분에 자연어 입력, 모델 추론, 샌드박스 실행, 결과 렌더링, 결제 흐름을 서로 분리하면서도 하나의 제품 경험으로 연결할 수 있었다.

## 핵심 구현 하이라이트

- 자연어 설명을 와이어프레임과 프로토타입 초안으로 연결하는 생성 흐름을 end to end로 구현
- Vercel AI SDK 기반 다중 LLM 지원 구조를 설계해 모델 선택 폭과 실험 유연성을 확보
- Excalidraw 렌더링을 제품 UI와 직접 연결해 생성 결과를 즉시 검토할 수 있게 구성
- E2B SDK 기반 샌드박스 실행 환경을 붙여 생성 결과의 실행 안정성과 격리를 확보
- Supabase, Upstash, Vercel KV를 조합해 세션 상태와 캐시를 분리 관리하는 서비스 구조 설계
- Stripe 결제 시스템을 통합해 프로토타입 도구를 실제 과금 가능한 제품 형태로 확장
