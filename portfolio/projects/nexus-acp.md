# Nexus ACP

터미널 중심 AI 에이전트 개발 환경을 네이티브 GUI 워크스페이스로 전환한 개발 도구

## 개요

Nexus ACP는 Claude Code를 포함한 AI 에이전트 기반 개발 환경을 터미널 중심 사용성에서 벗어나 시각적 워크스페이스로 재구성한 Tauri 기반 개발 도구다. 단순한 프론트엔드 셸이 아니라 세션 관리, 코드 편집, 프로토콜 연동, 로컬 앱 배포, 서버 데이터 계층까지 함께 다루는 제품으로 설계했다. 리포지토리 기준 개발 기간은 2025년 6월 19일부터 2026년 1월 6일까지이며, 총 400 commits가 누적되었다. GUI 전환 도구이면서 동시에 네이티브 앱 구조와 데이터 운영 체계까지 정비한 고밀도 개발 프로젝트다.

## 클라이언트 과제

AI 에이전트 개발 도구는 강력하지만 대부분 터미널과 텍스트 로그 중심으로 설계되어 있다. 이 방식은 숙련된 개발자에게는 효율적일 수 있지만, 세션 상태 추적, 명령 결과 비교, 파일 컨텍스트 확인, 코드 수정 같은 작업을 한눈에 다루기 어렵게 만든다. 클라이언트가 필요로 한 것은 CLI의 강점을 버리지 않으면서도 Claude Code 같은 에이전트 환경을 GUI로 풀어내는 작업 공간이었다. 여기에 더해 장기 운영을 위해 로컬 앱 레이어와 서버 레이어를 분리하고, 기존 SQLite 중심 흐름을 PostgreSQL 기반 구조로 마이그레이션하는 과제도 함께 있었다.

## 솔루션

이 프로젝트에서는 React, TypeScript, Vite 기반 프론트엔드에 Radix UI와 Tailwind를 결합해 세션 중심 인터페이스를 설계하고, Monaco를 붙여 코드 편집과 결과 확인을 같은 화면 안에서 처리할 수 있게 했다. 데스크톱 런타임은 Tauri로 구성해 웹 기술 스택의 생산성을 유지하면서도 네이티브 앱 형태로 배포할 수 있게 했고, ACP 연동 계층을 통해 AI 에이전트 상호작용을 GUI 안에서 다룰 수 있도록 설계했다. 서버 측에서는 PostgreSQL 기반 저장 구조를 정비하고 SQLite에서 PostgreSQL로의 마이그레이션 흐름을 정리해 데이터 계층의 확장성과 운영 안정성을 높였다. 결과적으로 이 프로젝트는 터미널 기반 AI 개발 경험을 더 넓은 사용자층이 다룰 수 있는 네이티브 개발 도구로 전환한 사례다.

## 아키텍처

리포지토리는 `src/`, `src-tauri/`, `server/`, `docs/`, `scripts/` 구조를 중심으로 나뉜다. `src/`는 세션 뷰, 작업 패널, 코드 편집 인터페이스 같은 사용자 경험 레이어를 담당하고, `src-tauri/`는 네이티브 앱 패키징과 시스템 연동을 처리하는 런타임 계층이다. `server/`는 ACP 기반 상호작용을 보조하는 서버 기능과 데이터 저장 계층을 담당하며, PostgreSQL 중심 구조로 마이그레이션된 백엔드 흐름을 담는다. `docs/`는 프로토콜 사용법, 제품 방향, 개발 규칙을 정리하는 문서 레이어이고, `scripts/`는 빌드와 데이터 관련 작업을 자동화하는 보조 계층이다. 전체적으로는 프론트엔드, 네이티브 런타임, 서버, 문서, 자동화 스크립트를 분리해 데스크톱 개발 도구로서의 유지보수성과 확장성을 확보한 구조다.

## 핵심 구현 하이라이트

- Claude Code를 포함한 AI 에이전트 개발 환경을 GUI 워크스페이스로 전환하는 제품 포지셔닝을 구체화
- React, Radix UI, Monaco 조합으로 세션 탐색과 코드 편집을 한 화면 안에서 다룰 수 있게 설계
- Tauri 기반 네이티브 앱 구조를 적용해 웹 기술 생산성과 데스크톱 배포 경험을 함께 확보
- ACP 연동 계층을 통해 터미널 중심 프로토콜 상호작용을 시각적 인터페이스 안으로 끌어옴
- SQLite에서 PostgreSQL로의 마이그레이션을 정리해 데이터 지속성과 장기 운영 기반을 강화
- `src/`, `src-tauri/`, `server/`, `docs/`, `scripts/` 레이어를 분리해 개발 도구 구조를 명확히 유지
